OpenScreenShot Docs

Support & FAQ

Answers to common questions, known limitations, and how to get help.

Frequently asked questions

Is my data sent to a server?

No. Zero data leaves your device. All capture, annotation, and export processing runs locally in your browser. The extension has no analytics, no telemetry, and no third-party services. See the Privacy Policy for details.

Does it work on chrome:// and extension pages?

No. For security reasons, Chrome blocks extensions from capturing chrome://, chrome-extension://, the Chrome Web Store, and about: pages. The extension will show an error message if you try.

Why does full-page capture take a few seconds?

Chrome limits captureVisibleTab to approximately 2 calls per second. Every viewport-height tile triggers one capture call, plus a short paint-settle delay. For a typical 10-tile page this takes about 5–6 seconds. The progress bar in the popup shows how it's going. Long pages take proportionally longer.

Why is my site header missing from the full-page screenshot?

Fixed headers do appear — they're captured on the first tile. If a header still seems missing, it may use position: sticky with certain scroll-behavior flags that the capture process handles differently. This is rare; if you encounter it, please open a GitHub issue.

Can I scroll while selecting a region?

Not yet. The region selection tool is currently viewport-only — it locks scrolling while you drag. Selecting an area that extends below the fold requires a full-page capture followed by cropping in the editor. Scroll-during-select is a planned feature.

How do I open the editor after a capture?

The editor opens automatically in a new tab after every capture — you don't need to do anything. If the popup is still open, it will close once the editor tab is ready. Export (download) your annotated screenshot from the editor's "Export" button in the top bar.

Can I use the editor without capturing a new screenshot?

Not directly — the editor loads the most recent capture from storage. If you want to edit an existing screenshot file, you'd need to capture it fresh through the extension. Loading arbitrary image files into the editor is a planned improvement.

Why does PDF export bundle extra libraries?

The extension uses jsPDF (lazy-loaded) for PDF generation. jsPDF transitively includes html2canvas and dompurify for its doc.html() feature, which we never call — we only use doc.addImage(). These libraries are bundled in the extension package but only downloaded when you export a PDF (lazy chunk). The total on-demand PDF cost is about 580 KB (gzipped ~180 KB).

Known limitations

  • Region selection is viewport-only — scrolling while selecting is not yet supported. Use full-page capture + crop instead.
  • Text and pen annotations are move-only — resize handles for text boxes and freehand strokes are planned.
  • Dynamic / infinite-scroll pages — the capture engine measures the page height once at the start. Pages that load content as-you-scroll may produce an incomplete result.
  • Editor i18n — the annotation editor UI is currently English only. The popup is fully internationalized.
  • No settings sync — your theme, export format, and filename template live in local storage and don't sync across devices or profiles.

Get help

If you hit a bug, have a feature request, or need help with anything not covered here:

  • Open a GitHub issue — github.com/pghqdev/OpenScreenShot/issues
  • Leave a review on the Chrome Web Store — it helps other users discover the extension.
  • Contribute — the extension is MIT licensed. Pull requests are welcome. See CONTRIBUTING.md.
