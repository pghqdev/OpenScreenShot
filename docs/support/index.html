Chrome limits captureVisibleTab to approximately 2 calls per second. Every viewport-height tile triggers one capture call, plus a short paint-settle delay. For a typical 10-tile page this takes about 5–6 seconds. The progress bar in the popup shows how it's going. Long pages take proportionally longer.

Fixed headers do appear — they're captured on the first tile. If a header still seems missing, it may use position: sticky with certain scroll-behavior flags that the capture process handles differently. This is rare; if you encounter it, please open a GitHub issue.

Currently the region selection tool is viewport-only — it locks scrolling while you drag. Selecting an area that extends below the fold requires a full-page capture followed by cropping in the editor. Scroll-during-select is a planned feature.

The extension uses jsPDF (lazy-loaded) for PDF generation. jsPDF transitively includes html2canvas and dompurify for its doc.html() feature, which we never call — we only use doc.addImage(). These libraries are bundled in the extension package but only downloaded when you export a PDF (lazy chunk). The total on-demand PDF cost is about 580 KB (gzipped ~180 KB).

Known limitations

  • Region selection is viewport-only — scrolling while selecting is not yet supported. Use full-page capture + crop instead.
  • Text and pen annotations are move-only — resize handles for text boxes and freehand strokes are planned.
  • Dynamic / infinite-scroll pages — the capture engine measures the page height once at the start. Pages that load content as-you-scroll (infinite scroll, lazy images) may produce an incomplete result. A single capture step is planned for the future.
  • Editor i18n — the annotation editor UI is currently English only. The popup is fully internationalized.
  • No settings sync — your theme, export format, and filename template live in local storage and don't sync across devices/browser profiles.
